# LINE予約システム構築

BiPシーシャの予約をLINEで完結させるための設計ガイド。貸切制・完全予約制のシーシャスペースに最適化した予約フローを構築する。

## 1. LINE公式だけで予約を受ける方法

LINE公式アカウントには標準の「予約機能」は搭載されていない。以下の3つのアプローチがある:

| 方法 | コスト | 自動化度 | 推奨度 |
|------|--------|---------|--------|
| **A. トークで手動受付** | 無料 | 低 | 開業初期向け |
| **B. 外部予約ツール連携** | 月額0〜5,000円 | 中〜高 | 安定運用向け |
| **C. Messaging API でBot構築** | 開発コストのみ | 高 | 技術力がある場合 |

---

## 2. 方法A: トーク手動受付（開業初期推奨）

### 2-1. 予約フロー

```
友だち追加
  ↓
リッチメニュー「予約」タップ
  ↓
自動応答で予約テンプレートを返信
  ↓
ユーザーがテンプレに沿って希望を送信
  ↓
運営者が手動で空き確認 → 確認メッセージ返信
  ↓
予約確定メッセージ送信
  ↓
前日リマインドメッセージ（手動 or ステップ配信）
```

### 2-2. 自動応答テンプレート

「予約」キーワードに対する自動応答メッセージ:

```
📅 ご予約ありがとうございます！
以下の内容を送ってください 👇

① 希望日:
② 希望時間帯: （14:00〜23:00の間）
③ 利用時間: （1時間 / 2時間 / 3時間）
④ 人数:
⑤ その他ご要望:

確認後、こちらから返信します！
（通常30分以内に返信します）
```

### 2-3. 予約確定メッセージテンプレート

```
✅ ご予約確定しました！

📅 日時: ○月○日（○）○:00〜○:00
👥 人数: ○名
💰 料金目安: ○,○○○円〜

📍 場所: [住所]
🗺 Googleマップ: [URL]

⚠️ キャンセル・変更は前日21時までにご連絡ください。

当日お会いできるのを楽しみにしています！
```

### 2-4. 手動受付の運用ルール

| ルール | 内容 |
|--------|------|
| 返信目標 | 30分以内（営業時間中） |
| 営業時間外 | 自動応答で「翌営業日に返信します」と案内 |
| 予約管理 | Googleカレンダーに手動登録 |
| ダブルブッキング防止 | 予約受付 → カレンダー確認 → 確定の順を徹底 |
| リマインド | 前日18:00に手動送信 |

## 3. 方法B: 外部予約ツール連携

友だち数が増えて手動対応が厳しくなったら、外部ツールとの連携を検討する。

### 3-1. おすすめツール比較

| ツール | 月額 | LINE連携 | 特徴 |
|--------|------|---------|------|
| **STORES予約** | 無料〜 | リンク連携 | 無料プランあり。予約ページURLをリッチメニューに設置 |
| **リピモ** | 1,100円 | ネイティブ連携 | LINE特化。Googleカレンダー連携可 |
| **tol（トル）** | 無料〜 | リンク連携 | スマホだけで予約管理。個人事業主向け |
| **Liny** | 5,000円〜 | 高度連携 | LINE公式の拡張ツール。セグメント配信も可能 |

### 3-2. STORES予約での連携手順（無料で始められる）

1. STORES予約に登録（無料プラン）
2. 予約タイプを「貸切」で作成
3. 予約メニューを設定:
   - 1時間コース / 2時間コース / 3時間コース
   - 人数選択
   - 料金設定
4. 予約ページURLを取得
5. LINE公式のリッチメニュー「予約」ボタンにURLを設定
6. 予約が入るとメール通知 → Googleカレンダーに自動反映

### 3-3. 連携時のリッチメニュー設計

```
┌─────────────────────┬──────────┐
│                     │  💰 料金  │
│   📅 今すぐ予約      ├──────────┤
│  （予約ページへ）     │  📍 場所  │
│                     ├──────────┤
│                     │  💬 問合せ │
└─────────────────────┴──────────┘
```

予約ボタンを最も大きく・目立つ位置に配置する。

## 4. 予約フロー全体設計

### 4-1. 理想的な顧客体験フロー

```
[認知] SNS・口コミ・チラシ
  ↓
[友だち追加] LINE友だち追加（特典あり）
  ↓
[あいさつ] 自動あいさつメッセージ + メニュー案内
  ↓
[予約] リッチメニューから予約
  ↓
[確認] 予約確定メッセージ（自動 or 手動）
  ↓
[リマインド] 前日リマインドメッセージ
  ↓
[来店] 来店・利用
  ↓
[フォロー] お礼メッセージ + レビュー依頼 + 次回クーポン
  ↓
[リピート] 定期的な情報配信 → 再予約
```

### 4-2. リマインドメッセージ

前日のリマインドは**キャンセル率を大幅に下げる**効果がある。

**前日リマインドテンプレート**:

```
🔔 明日のご予約リマインドです！

📅 ○月○日（○）○:00〜○:00
👥 ○名様
📍 [住所]

お気をつけてお越しください！

⚠️ キャンセル・変更はこのトークでご連絡ください。
```

### 4-3. ステップ配信でリマインドを半自動化

LINE公式のステップ配信機能を使えば、予約タグを付けたユーザーに対して自動でリマインドを送れる（ただし日時指定が柔軟でないため、完全自動化は外部ツールかAPI連携が必要）。

## 5. 予約管理の効率化

### 5-1. Googleカレンダーでの管理

1人運営では高機能な予約システムより、Googleカレンダーでシンプルに管理するのが効率的:

| 項目 | 設定 |
|------|------|
| カレンダー名 | 「BiPシーシャ 予約」 |
| 予約イベント | タイトルに「○○様 ○名」、メモにLINE名・連絡先 |
| 色分け | 確定=青、仮予約=黄、キャンセル=赤 |
| 通知 | 前日+当日朝にリマインド通知 |

### 5-2. 予約台帳テンプレート（Googleスプレッドシート）

| 予約日 | 時間 | 名前 | LINE名 | 人数 | コース | 料金 | ステータス | 備考 |
|--------|------|------|--------|------|--------|------|-----------|------|
| 3/25 | 18:00-20:00 | 田中 | @tanaka | 3名 | 2時間 | ¥6,000 | 確定 | 初回 |

### 5-3. キャンセルポリシー

大学生は急な予定変更が多いため、厳しすぎないポリシーが重要:

| タイミング | 対応 |
|-----------|------|
| 前日21:00まで | 無料キャンセル |
| 当日 | 原則キャンセル料50%（相談可） |
| 無断キャンセル | 次回予約時に前払い |

> **1人運営のコツ**: 無断キャンセルが続くユーザーにはタグを付けて管理。柔軟対応しつつも、ルールは明示しておく。

| フェーズ | 友だち数目安 | 予約方式 | 月額コスト |
|---------|------------|---------|-----------|
