# BIPコンテンツの作り方

## 1. 公開すべき情報と非公開にすべき情報の線引き

### 公開する情報

BIPの原則は「迷ったら公開」。ただし、以下のフレームワークで判断する。

| カテゴリ | 公開する情報 | 理由 |
|---------|------------|------|
| **数字** | 原価率、家賃、初期投資額、月次売上目標 | 数字の透明性がBIPの核。信頼を生む |
| **意思決定** | なぜこの物件を選んだか、なぜこの料金にしたか | 「考え方」を見せることで共感と学びを提供 |
| **失敗** | 物件に断られた、許認可で問題が出た、見積もりが想定の2倍だった | 失敗コンテンツはエンゲージメントが最も高い |
| **プロセス** | AIとの対話ログ、調査の過程、比較検討の表 | BIPの最大の差別化。他のシーシャ店には出せない |
| **学び** | 調べて分かったこと、知らなかったこと | 教育コンテンツとしての価値 |

### 非公開にする情報

| カテゴリ | 非公開にする情報 | 理由 |
|---------|----------------|------|
| **個人のプライバシー** | 自宅住所、銀行口座、家族の情報 | 安全上の理由 |
| **契約の具体的条件** | オーナーとの交渉の詳細、契約書の全文 | 相手方のプライバシー保護 |
| **他者の個人情報** | 不動産業者の担当者名、取引先の内部情報 | 信頼関係の維持 |
| **セキュリティ情報** | 鍵の管理方法、防犯カメラの配置 | 安全上の理由 |
| **リアルタイムの銀行残高** | 手元資金の正確な額 | 個人の財務情報。範囲（「初期投資100万円以内を目標」等）なら公開OK |

### グレーゾーンの判断基準

「公開して誰かが傷つくか？」がNOなら公開。YESなら非公開。

## 2. ストーリーテリングの技法

### 共感 → 応援 → 参加のファネル

BIPコンテンツの最終目標は「来店」だが、そこに至るまでに3つのステージがある。

```
共感: 「あ、この人の気持ちわかる」
  ↓
応援: 「頑張ってほしい。気になる」
  ↓
参加: 「行ってみたい。友達も誘おう」
```

各ステージで出すべきコンテンツが異なる。

### ステージ1: 共感（認知〜興味）

**目的**: 「この人、面白いな」と思わせる

| コンテンツ例 | なぜ共感を生むか |
|------------|----------------|
| 「東京でAIに仕事を奪われて岩手に帰ってきた」 | 挫折からの再起。人間味がある |
| 「盛岡にシーシャ屋がなくて困ってる」 | 同じ不便を感じている人がいる |
| 「大学生が一人で事業を始めようとしている」 | 応援したくなる |
| 「好きだからやる。それ以上の理由は要らない」 | 率直さに共感 |

### ステージ2: 応援（興味〜関心）

**目的**: 「この人を応援したい」と思わせる

| コンテンツ例 | なぜ応援を生むか |
|------------|----------------|
| 「物件を5件内見して全部ダメだった」 | 苦労しているのが伝わる |
| 「保健所に電話したら想定外の許認可が必要だった」 | リアルな壁。一緒にハラハラする |
| 「原価を全部計算したら意外と利益率が高い」 | 「おっ、いけるかも」と一緒にワクワク |
| 「AIと一緒にサイトを作ったらこうなった」 | 技術的な面白さ |

### ステージ3: 参加（関心〜行動）

**目的**: 「行きたい」と思わせる

| コンテンツ例 | なぜ参加を生むか |
|------------|----------------|
| 「物件決まりました！内装工事始めます」 | 現実味が出る。カウントダウンの始まり |
| 「内装の進捗。今こんな感じ」 | ビフォーアフターは最強コンテンツ |
| 「オープン日決定。最初の1週間は友達限定で」 | 限定感。早く行きたい |
| 「プレオープンで来てくれた人の反応」 | 社会的証明。他の人も楽しんでる |

### ストーリーテリングの型

#### 「Before → Struggle → After」

```
Before: 東京から盛岡に帰ってきた。シーシャ屋がない。
Struggle: 物件探し、許認可、資金調達...壁だらけ。
After: ついにオープン。岩大生が集まる場所ができた。
```

この大きなアークの中に、日々の小さな「Before → Struggle → After」を積み重ねる。

#### 「数字 + 感情」の組み合わせ

数字だけだと冷たい。感情だけだとふわっとする。両方を組み合わせる。

- NG: 「原価率は15%です」（冷たい）
- NG: 「原価計算して安心しました」（ふわっとしてる）
- OK: 「原価率15%。正直、飲食でこんなに利益率高い業態があるとは思わなかった。これならいける」

## 3. 事業構築の各フェーズで出すべきコンテンツ

## 4. SNS × Webサイト × メディア露出の連動

### 三層構造

```
SNS（Instagram）: フロー型。日々の更新。感情・ビジュアル中心
     ↕ 相互リンク
Webサイト: ストック型。構造化された情報。数字・分析中心
     ↕ ネタ提供
メディア: 第三者による信頼の付与。リーチの拡大
```

### 各媒体の役割

| 媒体 | 役割 | コンテンツの特徴 |
|------|------|-----------------|
| **Instagram** | 日々の接触点。認知獲得 | ビジュアル重視。短い文章。Storiesで日常、投稿で節目 |
| **Webサイト** | BIPの本体。深い情報のストック | 数字、分析、比較表。サイト自体がBIPのプロダクト |
| **地方メディア** | 信頼の増幅。広いリーチ | 取材記事。第三者の視点で語られることで信頼度UP |

### 連動のフロー

1. **何か進捗がある** → Instagram Storiesでリアルタイム共有
2. **まとまった情報になる** → Webサイトに構造化して掲載
3. **節目（物件決定、オープン等）** → Instagram投稿 + Webサイト更新 + プレスリリース
4. **メディアに取り上げられる** → そのメディア記事をInstagramでシェア → サイトに「メディア掲載」として追加

### Instagramの運用方針

| フォーマット | 用途 | 頻度 |
|------------|------|------|
| **Stories** | 日常の進捗、気づき、質問 | 毎日〜隔日 |
| **フィード投稿** | 節目の報告、ビジュアルコンテンツ | 週1〜2回 |
| **Reels** | 内装工事のタイムラプス、空間紹介 | 週1回（フェーズ3以降） |
| **ハイライト** | 「物件探し」「内装工事」「コンセプト」等カテゴリ別にまとめ | 随時 |

## 5. 失敗や挫折の公開方法

### なぜ失敗コンテンツが最強か

- **エンゲージメントが高い**: 成功報告より失敗談のほうが反応が多い
- **共感を生む**: 「自分も同じことで悩んだ」という共感
- **信頼性が上がる**: 良いことしか言わない人は信用されない
- **教育的価値**: 他の人が同じ失敗を避けられる

### 失敗の公開フレームワーク

```
1. 何が起きたか（事実）
2. なぜ起きたか（分析）
3. どう感じたか（感情）
4. 何を学んだか（学び）
5. 次にどうするか（行動）
```

### 具体例

#### 例: 物件に断られた場合

```
事実: 「上田の物件、シーシャ営業はNGと言われた」
分析: 「住居専用地域だったのと、オーナーが火気使用を嫌がった」
感情: 「正直めちゃくちゃ落ち込んだ。ここが一番岩大に近かったのに」
学び: 「用途地域の確認は内見前にやるべき。オーナー交渉には安全対策資料が必須」
行動: 「次は近隣商業地域に絞って探す。安全対策の資料を作ってから内見に行く」
```

#### 例: 想定外の費用が発覚した場合

```
事実: 「換気工事の見積もりが150万円。想定の2倍」
分析: 「スケルトン物件の場合、ダクトをゼロから作る必要があった」
感情: 「予算オーバー。正直、一回心が折れた」
学び: 「居抜き物件を探すべき。既存のダクトがあれば半額以下」
行動: 「アットホームで居抜き物件を再検索。バー・スナックの居抜きに絞る」
```

### 失敗公開のNG

- **他人を晒す**: 「あの不動産屋は対応が最悪だった」→ 絶対NG。自分の失敗として語る
- **愚痴で終わる**: 学びと次のアクションがないと、ただの愚痴になる
- **盛る**: 失敗を大げさにするのは信頼を損なう。事実をそのまま
- **リアルタイムすぎる感情の吐露**: 落ち込んでいるときに書くと後悔する。1日置いてから書く

## コンテンツカレンダーのテンプレート

### 週次の投稿計画

| 曜日 | 媒体 | 内容 |
|------|------|------|
| 月 | Instagram Stories | 今週やること宣言 |
| 火 | — | 作業日（コンテンツ制作なし） |
| 水 | Instagram投稿 | 週の学び・進捗まとめ |
| 木 | — | 作業日 |
| 金 | Instagram Stories | 今週の振り返り |
| 土 | Webサイト更新 | 週の成果を構造化してサイトに反映 |
| 日 | — | 休み |

> これはあくまでテンプレート。無理に全部やる必要はない。「更新頻度 > 完成度」の原則を忘れない。
