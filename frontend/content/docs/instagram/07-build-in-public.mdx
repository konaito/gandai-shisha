## Build in Publicとは

プロダクトや事業の構築プロセスを透明に公開しながら進める手法。完成品だけでなく、開発の裏側・意思決定・成功も失敗もリアルタイムで共有し、オーディエンスと共に事業を育てていくアプローチ。

### 起源
- Buffer CEO Joel Gascoigneが先駆者。2013年に全社員の給与を公開
- Kevon Cheungが「Build in Public Definitive Guide」を執筆、6,000人以上に影響
- #buildinpublic がX（Twitter）で定着

### なぜ今注目されているか
- SNSで個人が直接オーディエンスにリーチ可能に
- インディーハッカー・ソロプレナーの台頭
- 消費者が「ブランドの裏側」や「人間味」を求める時代

## 成功事例

### テック・SaaS
| 事例 | 成果 |
|------|------|
| Buffer | 全社員給与・収益公開。Forbes等に掲載。「Open Startup」運動の火付け役 |
| Pieter Levels | Nomad List・Remote OKを完全BIPで構築。数百万ドル規模に |
| Tony Dinh (Xnapper) | 1ツイートで1,700いいね、月収$6,000 |
| 匿名ソロプレナー | 90日で月間経常収益$62,000 |

### 実店舗
| 事例 | 成果 |
|------|------|
| Stuckey's | ロードサイド小売チェーンの再建過程を公開。6ヶ月で黒字化 |
| Rosa's Thai Cafe | 屋台→UK18店舗、年間売上$25M超 |
| NO COFFEE（福岡） | 開業1ヶ月前から発信。現在84,000フォロワー |

## 共有すべきコンテンツ

1. **進捗報告**: WIPのスクリーンショット、ビフォーアフター
2. **ビジネス指標**: 売上推移、ユーザー数、マイルストーン
3. **失敗談・学び**: うまくいかなかったこと、そこからの学び
4. **意思決定の裏側**: なぜその方向に進んだか、他の選択肢
5. **裏側の様子**: チームの働き方、日常のワークフロー

## メリット

- **信頼構築**: 苦労も見せることで親近感
- **早期フィードバック**: コストがかかる前に盲点を発見
- **コミュニティ形成**: 参加している感覚→ロイヤルな支持者
- **採用力向上**: 哲学や進捗の公開が人材を引きつける
- **マーケティングコスト削減**: 無料でメディア掲載
- **アカウンタビリティ**: 公開が自分へのコミットメントに

## デメリット

- 競合リスク（情報がステルスモードの競合に利用される可能性）
- 時間と注意力の分散
- 精神的プレッシャー（進捗が遅い時）
- 低質なフィードバックに振り回されるリスク

## 透明性の3段階設計（信号機モデル）

| レベル | 色 | 共有内容 |
|--------|-----|---------|
| 安全 | 緑 | 一般的な学び、進捗、業界の知見 |
| 要確認 | 黄 | 価格テスト、ロードマップ変更、機能ロールバック |
| 非公開 | 赤 | セキュリティ詳細、顧客データ、ベンダー契約 |

### 収益数字の公開ベストプラクティス
- 初期段階: 具体的な数字OK（注目を集めやすい）
- 成長段階: 丸めた数字やパーセンテージ成長に
- 成熟段階: トレンドやマイルストーンの共有にとどめる

## 避けるべきこと

1. 都合のいい数字だけ見せる → 信頼を損なう
2. 表面的な失敗共有（本質的な痛みを見せない）
3. 過剰共有（競争優位を失う）
4. 一貫性の欠如（苦しい時だけ沈黙する）
5. フィードバックに振り回されて方向性を見失う

## ストーリーテリング

### BIPストーリーテリングの3本柱
1. **真正性**: 作り込まれた成功談ではなくリアルな過程
2. **感情的訴求**: 喜び・苦悩・恐れ・達成感を率直に
3. **シンプルさ**: 誰でも理解できる言葉

### 「応援したくなる」ストーリーの4要素
1. **共感できる主人公**: オーナー自身。動機・恐れ・課題を率直に
2. **本物感**: リアルな人のリアルな課題
3. **テンション→変化**: 課題→克服→成長
4. **参加**: フォロワーが意見を言え、選択に関われる
